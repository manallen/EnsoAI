# 域自适应问题模板

根据项目域类型（UI/后端/数据）提供差异化的探索问题。

## 通用必问问题

这些问题适用于所有类型的项目，优先级最高：

| ID | 问题 | 目的 |
|----|------|------|
| G-1 | 项目的核心目标用户是谁？他们的技术水平如何？ | 用户画像 |
| G-2 | 这个项目需要解决的核心问题是什么？ | 核心价值 |
| G-3 | 项目有预设的技术栈或框架偏好吗？ | 技术约束 |
| G-4 | 项目的规模预期是什么？（MVP / 正式产品 / 大型系统） | 范围控制 |
| G-5 | 有哪些已知的时间或资源限制？ | 约束识别 |

## UI-Heavy 项目问题

当项目检测为 UI 为主时，使用以下问题：

### 高优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| UI-1 | 需要支持哪些设备和屏幕尺寸？（桌面/平板/移动端/响应式） | 设备适配 |
| UI-2 | 对组件库有偏好吗？（自建/Ant Design/Material UI/其他） | 组件选择 |
| UI-3 | 需要支持深色模式或主题切换吗？ | 主题需求 |
| UI-4 | 状态管理的复杂度如何？（简单本地状态/全局状态管理） | 状态管理 |
| UI-5 | 对交互动画有特殊要求吗？（基础过渡/复杂动画/3D效果） | 动画需求 |

### 中优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| UI-6 | 需要支持国际化(i18n)吗？ | 国际化 |
| UI-7 | 对可访问性(a11y)有要求吗？（WCAG 等级） | 无障碍 |
| UI-8 | 需要离线支持或 PWA 功能吗？ | 离线能力 |
| UI-9 | 对首屏加载时间有明确要求吗？ | 性能指标 |
| UI-10 | 需要支持 SSR 或 SSG 吗？ | 渲染模式 |

### 低优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| UI-11 | 有品牌设计规范或设计系统需要遵循吗？ | 设计约束 |
| UI-12 | 需要集成哪些第三方组件？（地图/图表/富文本编辑器） | 第三方集成 |

## Backend-Heavy 项目问题

当项目检测为后端为主时，使用以下问题：

### 高优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| BE-1 | API 设计风格偏好？（REST/GraphQL/gRPC/其他） | API 风格 |
| BE-2 | 认证和授权方案是什么？（JWT/OAuth/Session/无需认证） | 安全方案 |
| BE-3 | 数据库类型偏好？（关系型/文档型/键值/图数据库） | 数据存储 |
| BE-4 | 预期的并发用户数量级？（个位数/百/千/万+） | 扩展性 |
| BE-5 | 需要处理事务吗？复杂度如何？ | 事务需求 |

### 中优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| BE-6 | 需要缓存策略吗？（无/应用级/分布式） | 缓存 |
| BE-7 | 需要消息队列或异步任务处理吗？ | 异步处理 |
| BE-8 | 对 API 响应时间有明确要求吗？ | 性能指标 |
| BE-9 | 需要集成哪些外部服务？（支付/邮件/SMS/云存储） | 集成需求 |
| BE-10 | 日志和监控的需求是什么？ | 可观测性 |

### 低优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| BE-11 | 部署环境是什么？（本地/云/容器/Serverless） | 部署方案 |
| BE-12 | 需要考虑多租户架构吗？ | 架构模式 |

## Data-Heavy 项目问题

当项目检测为数据为主时，使用以下问题：

### 高优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| DA-1 | 数据来源是什么？（API/数据库/文件/网页爬取/实时流） | 数据来源 |
| DA-2 | 数据处理频率是什么？（实时/近实时/批处理/按需） | 处理频率 |
| DA-3 | 输出格式要求是什么？（API/文件/数据库/报表） | 输出格式 |
| DA-4 | 数据量级预期是多少？（MB/GB/TB+） | 数据规模 |
| DA-5 | 对数据质量有什么要求？（去重/校验/清洗规则） | 数据质量 |

### 中优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| DA-6 | 需要支持增量处理还是全量处理？ | 处理模式 |
| DA-7 | 需要断点续传或失败重试机制吗？ | 容错能力 |
| DA-8 | 对处理延迟有明确要求吗？ | 时效性 |
| DA-9 | 数据需要存储多久？有归档策略吗？ | 数据生命周期 |
| DA-10 | 需要数据血缘追踪吗？ | 数据治理 |

### 低优先级

| ID | 问题 | 关注点 |
|----|------|--------|
| DA-11 | 需要可视化 Dashboard 吗？ | 数据展示 |
| DA-12 | 有数据安全或合规要求吗？（GDPR/脱敏/加密） | 数据安全 |

## 问题选择策略

### 问题队列生成规则

```
1. 始终包含通用必问问题 (G-1 ~ G-5)
2. 根据域偏重添加对应域的高优先级问题
3. 如果是混合型项目，按权重比例混合问题
4. 问题总数控制在 8-12 个（首轮）
```

### 示例：UI-Heavy 项目

```
问题队列:
  1. [G-1] 项目的核心目标用户是谁？他们的技术水平如何？
  2. [G-2] 这个项目需要解决的核心问题是什么？
  3. [UI-1] 需要支持哪些设备和屏幕尺寸？
  4. [UI-2] 对组件库有偏好吗？
  5. [G-3] 项目有预设的技术栈或框架偏好吗？
  6. [UI-3] 需要支持深色模式或主题切换吗？
  7. [UI-4] 状态管理的复杂度如何？
  8. [G-4] 项目的规模预期是什么？
```

### 示例：后端为主 + 数据辅助

```
问题队列:
  1. [G-1] 项目的核心目标用户是谁？
  2. [G-2] 这个项目需要解决的核心问题是什么？
  3. [BE-1] API 设计风格偏好？
  4. [BE-2] 认证和授权方案是什么？
  5. [DA-1] 数据来源是什么？
  6. [BE-3] 数据库类型偏好？
  7. [DA-2] 数据处理频率是什么？
  8. [G-4] 项目的规模预期是什么？
  9. [BE-4] 预期的并发用户数量级？
  10. [DA-4] 数据量级预期是多少？
```

## 深挖问题

当用户选择"需要更多信息"时，从中优先级问题中选择：

```
触发条件: 用户选择 [C] 需要更多信息

动作:
  1. 从当前域的中优先级问题中选择 2-4 个
  2. 添加到问题队列
  3. 继续单问题确认流程
```
