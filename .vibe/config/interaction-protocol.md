# 单问题交互协议

定义 vibe:explore 阶段的用户交互格式和问题队列管理规则。

## 问题呈现格式

### 标准问题卡片

```
┌─────────────────────────────────────────────────────────────────────┐
│  问题 X/N (轮次 Y/3)                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  [问题ID] 问题内容                                                  │
│                                                                     │
│  背景: (可选) 为什么这个问题重要                                    │
│                                                                     │
│  ─────────────────────────────────────────────────────────────      │
│                                                                     │
│  选项:                                                              │
│    [A] 选项 A 描述                                                  │
│    [B] 选项 B 描述                                                  │
│    [C] 需要更多信息 (触发深挖)                                      │
│    [N] 无补充 / 跳过此问题                                          │
│                                                                     │
│  ─────────────────────────────────────────────────────────────      │
│                                                                     │
│  快捷操作:                                                          │
│    [F] 强制继续 — 跳过剩余问题，使用默认值                          │
│    [Q] 查看进度 — 显示问题队列状态                                  │
│    [R] 回顾 — 查看已回答的问题                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 进度指示器

```
问题 X/N: 当前是第 X 个问题，本轮共 N 个问题
轮次 Y/3: 当前是第 Y 轮深挖，最多 3 轮
```

## 用户选项格式

### 标准选项

| 选项 | 含义 | 处理 |
|------|------|------|
| [A], [B], [C]... | 具体选项 | 记录选择，继续下一问题 |
| [N] | 无补充 | 记录为"无补充"，计入连续计数 |
| [F] | 强制继续 | 跳过剩余问题，使用默认值，退出循环 |
| [Q] | 查看进度 | 显示问题队列状态，不影响流程 |
| [R] | 回顾 | 显示已回答问题，不影响流程 |

### 自定义输入

用户可以直接输入文字而非选择选项：

```
输入: "我想要支持移动端和桌面端，但移动端优先"

处理:
  1. 识别为自定义输入
  2. 记录原始回答
  3. 提取关键约束: "移动端优先，支持桌面端"
  4. 继续下一问题
```

## 问题队列管理

### 队列数据结构

```yaml
question_queue:
  round: 1                    # 当前轮次
  total_rounds: 3             # 最大轮次
  questions:
    - id: "G-1"
      text: "项目的核心目标用户是谁？"
      status: "answered"      # pending | answered | skipped
      answer: "开发者"
      selected_option: "A"
    - id: "UI-1"
      text: "需要支持哪些设备？"
      status: "pending"
      answer: null
      selected_option: null
  consecutive_no_supplement: 0  # 连续"无补充"计数
  total_questions_asked: 5      # 累计已问问题数
```

### 队列操作规则

| 操作 | 触发条件 | 动作 |
|------|----------|------|
| 添加问题 | 用户选择 [C] 需要更多信息 | 从中优先级问题中选择 2-4 个加入队列 |
| 跳过问题 | 用户选择 [N] | 标记为 skipped，consecutive_no_supplement++ |
| 重置连续计数 | 用户选择非 [N] 选项 | consecutive_no_supplement = 0 |
| 清空队列 | 用户选择 [F] | 剩余问题标记为 skipped，退出循环 |

## "无补充"连续计数逻辑

### 计数规则

```
当用户选择 [N] 时:
  consecutive_no_supplement++

当用户选择其他选项时:
  consecutive_no_supplement = 0

退出条件检查:
  if consecutive_no_supplement >= 2:
    触发退出条件 (连续2问无补充)
```

### 示例流程

```
Q1: 用户选择 [A] → consecutive = 0
Q2: 用户选择 [N] → consecutive = 1
Q3: 用户选择 [B] → consecutive = 0
Q4: 用户选择 [N] → consecutive = 1
Q5: 用户选择 [N] → consecutive = 2 → 触发退出
```
