# 退出条件控制

定义 vibe:explore 阶段递进式深挖的退出条件和处理流程。

## 退出条件定义

### 4 种退出条件

| ID | 条件 | 阈值 | 类型 |
|----|------|------|------|
| EXIT-1 | 达到最大轮次 | round >= 3 | 硬退出 |
| EXIT-2 | 达到问题上限 | total_questions >= 12 | 硬退出 |
| EXIT-3 | 连续无补充 | consecutive_no_supplement >= 2 | 软退出 |
| EXIT-4 | 用户强制继续 | user_signal == "force_continue" | 用户退出 |

### 退出类型说明

- **硬退出**: 无条件退出，不可继续
- **软退出**: 提示用户，可选择继续
- **用户退出**: 用户主动触发

## 退出条件检查逻辑

### 检查时机

```
每个问题回答后:
  1. 更新状态 (consecutive_no_supplement, total_questions)
  2. 检查退出条件
  3. 如果满足任一条件，触发退出流程
  4. 否则继续下一问题
```

### 检查优先级

```
优先级从高到低:
  1. EXIT-4 (用户强制) - 立即退出
  2. EXIT-1 (最大轮次) - 硬退出
  3. EXIT-2 (问题上限) - 硬退出
  4. EXIT-3 (连续无补充) - 软退出，提示确认
```

### 伪代码

```python
def check_exit_conditions(state):
    # 用户强制退出
    if state.user_signal == "force_continue":
        return {"exit": True, "reason": "EXIT-4", "type": "user"}

    # 最大轮次
    if state.round >= 3:
        return {"exit": True, "reason": "EXIT-1", "type": "hard"}

    # 问题上限
    if state.total_questions >= 12:
        return {"exit": True, "reason": "EXIT-2", "type": "hard"}

    # 连续无补充
    if state.consecutive_no_supplement >= 2:
        return {"exit": True, "reason": "EXIT-3", "type": "soft"}

    return {"exit": False}
```

## "强制继续"后的行为

当用户在探索过程中选择 [F] 强制继续后，如果后续触发软退出 (EXIT-3) 确认时选择"继续深挖"：

### 规则

```
触发 EXIT-4 后:
  - 不再检查 EXIT-1 (最大轮次)
  - 仍然检查 EXIT-2 (问题上限 12) - 作为最终保护
  - 仍然检查 EXIT-3 (连续无补充) - 但用户可继续选择"继续深挖"
```

### 状态标记

```yaml
exploration_state:
  force_continue_triggered: true  # 用户曾触发强制继续
  round_limit_bypassed: true       # 轮次限制已解除
```

## 退出后处理

### 约束集聚合

```
退出后:
  1. 汇总所有已回答问题
  2. 提取约束 (HC/SC)
  3. 标记待确认项 (TBD)
  4. 进入可视化呈现
```

### 待确认项处理

```yaml
待确认项来源:
  - 用户选择 [N] 跳过的问题
  - 用户选择 [F] 强制跳过的问题
  - 探索中发现但未深挖的问题

处理方式:
  - 标记为 [TBD-x]
  - 在需求文档中列出
  - 可在审查阶段补充
```
