## 检测规则

### 评分算法

```
domain_score = sum(keyword_weight) / keyword_count

判定规则:
1. 计算每个域的总权重
2. 归一化为百分比
3. 域偏重 = 主域(最高分) + 次域(次高分，如果 > 30%)
```

### 判定阈值

| 场景 | 规则 |
|------|------|
| 单域明显 | 主域得分 > 60%，其他 < 20% |
| 双域混合 | 主域得分 40-60%，次域 > 30% |
| 三域均衡 | 所有域得分在 25-40% 之间 |

### 示例

```
输入: "实现一个 React 组件库，支持响应式布局"

检测:
  UI:     React(1.0) + 组件(1.0) + 响应式(0.8) + 布局(0.8) = 3.6
  后端:   无匹配 = 0
  数据:   无匹配 = 0

结果: UI-Heavy (100%)
```

```
输入: "开发一个用户管理系统，包含登录注册、权限管理、数据统计"

检测:
  UI:     无明显匹配 = 0
  后端:   登录(0.8) + 权限(0.8) = 1.6
  数据:   统计(分析 0.8) = 0.8

结果: Backend-Heavy (67%) + Data (33%) = "后端为主 + 数据辅助"
```
