# UI Anti-Patterns

这份模板用于初始化 `.vibe/memory/patterns/ui-anti-patterns.md`，供 `vibe:design`、`vibe-ui-architect`、`vibe-reviewer` 复用。

## 视觉与色彩

- 禁止把霓虹、过饱和主色当成默认主视觉
- 禁止正文对比度过低
- 主色不宜过多，避免同层级竞争

## 排版与层级

- 禁止超过 2 种字体长期并用
- 禁止正文字号过小、行高过紧
- 禁止大段正文没有宽度控制

## 交互与可用性

- 禁止缺少 hover / focus / active / disabled 等关键状态
- 禁止错误地把不可点击元素也写成 `pointer`
- 禁止点击区明显小于常规触控或鼠标操作范围

## 组件与布局

- 禁止把 emoji 当成唯一功能图标来源
- 禁止容器没有基础内边距
- 禁止响应式布局只靠硬编码断点拼凑

## 可访问性

- 禁止只靠颜色传达状态或错误
- 禁止图片缺少可替代文本
- 禁止表单错误反馈只出现在颜色变化里
